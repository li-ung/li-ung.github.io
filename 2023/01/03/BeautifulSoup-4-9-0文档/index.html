Beautiful Soup 4.9.0 文档

原文档：Beautiful Soup Documentation

Beautiful Soup 是一个用于从 HTML 和 XML 文件中提取数据的 Python 库。它能够与你最喜欢的解析器一起，提供惯用的遍历、查找和修改解析树的方法。它通常可以为程序员节省数小时或数天的工作时间。

这篇文档通过示例说明了 Beautiful Soup 4 的所有主要功能。将会展示这个库的优点、工作原理、使用方法、如何让它做你想做的事，以及如何处理异常情况。

本文档涵盖 Beautiful Soup 4.11.0 版。 文档中的示例是用 Python 3.8 编写的。

你可能在寻找 Beautiful Soup 3 的文档，但是 Beautiful Soup 3 已经停止开发，并在 2020 年 12 月 31 日取消了所有支持。如果您想了解 Beautiful Soup 3 和 Beautiful Soup 4 之间的差异，请参阅将代码移植到 BS4。

Beautiful Soup 的安装

如果你使用的是最新版的 Debian 或 Ubuntu，可以使用系统包管理器安装 Beautiful Soup：

$ apt-get install python3-bs4

Beautiful Soup 4 是通过 PyPi 发布的，所以如果你无法用系统包管理器安装它，可以用 easy_install 或 pip 安装，包名是 beautifulsoup4。 确保你使用的 pip 或 easy_install 版本与你的 Python 版本匹配。（它们可能叫做 pip3 和 easy_install3）。

$ easy_install beautifulsoup4

$ pip install beautifulsoup4

（注意！有一个名为BeautifulSoup 包不是我们要安装的。那是以前使用的主要版本 BeautifulSoup 3。因为很多软件使用的是 BS3，所以它仍然可用。如果你要编写新代码，应该安装beautifulsoup4。）

如果你没有安装 easy_install 或 pip，可以下载 Beautiful Soup 4 源码压缩包 然后用 setup.py 安装。

$ python setup.py install

如果以上方法都失败了，Beautiful Soup 的发布协议允许你将整个 BS4 库打包在你的项目中。你可以下载 BS4 源码压缩包，将它的 bs4 目录复制到你的项目的代码库中，然后就可以使用 Beautiful Soup 而无需安装。

作者使用的是 Python 3.8 开发 Beautiful Soup，但它应该可以在其他新版本的 Python 中使用。

安装解析器

Beautiful Soup 支持 Python 标准库中包含的 HTML 解析器，也支持许多第三方 Python 解析器。 一个是 lxml 解析器。 根据不同的系统，可以选择以下命令来安装 lxml：

$ apt-get install python-lxml

$ easy_install lxml

$ pip install lxml

另一个选择是纯 Python 实现（Pure-Python）的 html5lib 解析器。它与 Web 浏览器用同样的方式解析 HTML。根据不同系统选择以下命令来安装 html5lib：

$ apt-get install python-html5lib

$ easy_install html5lib

$ pip install html5lib

下表总结了每个解析器库的优缺点：

解析器 使用方法 优势 劣势
Python标准库 BeautifulSoup(markup, "html.parser") 1.Python的内置标准库 2.执行速度适中 3.兼容性较好 不如lxml快，也没有html5lib的兼容性好
lxml HTML 解析器 BeautifulSoup(markup, "lxml") 1. 速度很快 2. 兼容性较好 需要C语言外部依赖
lxml XML 解析器 BeautifulSoup(markup, ["lxml-xml"])``BeautifulSoup(markup, "xml") 1. 速度很快 2. 当前唯一支持 XML 的解析器 需要C语言外部依赖
html5lib BeautifulSoup(markup, "html5lib") 1. 兼容性很强 2.以web浏览器的方式解析文档 3.能生成有效的HTML5格式的文档 1. 速度很慢 2. 需要 Python 外部依赖

作者建议安装并使用 lxml 以提高速度。 如果使用的Python 版本非常旧（早于 3.2.2），则必须安装 lxml 或 html5lib。 Python 的内置 HTML 解析器在旧版本中并不是很好。

请注意，如果要解析的文档格式错误，使用不同的解析器将生成不同的 Beautiful Soup 树。 详细信息请参阅解析器之间的差异。

安装完成后的问题

Beautiful Soup发布时打包成Python2版本的代码,在Python3环境下安装时,会自动转换成Python3的代码,如果没有一个安装的过程,那么代码就不会被转换.

如果代码抛出了 ImportError 的异常: “No module named HTMLParser”, 这是因为你在Python3版本中执行Python2版本的代码.

如果代码抛出了 ImportError 的异常: “No module named html.parser”, 这是因为你在Python2版本中执行Python3版本的代码.

如果遇到上述2种情况,最好的解决方法是重新安装BeautifulSoup4.

如果在ROOT_TAG_NAME = u’[document]’代码处遇到 SyntaxError “Invalid syntax”错误,需要将把BS4的Python代码版本从Python2转换到Python3. 可以重新安装BS4:

或在bs4的目录中执行Python代码版本转换脚本
