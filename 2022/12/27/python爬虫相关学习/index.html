python爬虫相关学习

一、http请求与响应

参考：Python爬虫详解（一看就懂）

http请求

请求报文包含：请求头，请求行，空行，请求数据

  • 请求行：

    • 请求方法，常见的请求方法有 GET、POST、PUT、DELETE、HEAD
    • 客户端要获取的资源路径
    • 是客户端使用的 HTTP 协议版本号

  • 请求头：客户端向服务器发送请求的补充说明，比如说明访问者身份

  • 请求体：客户端向服务器提交的数据，比如用户登录时需要提高的账号密码信息。请求体并不是所有的请求都有的，比如一般的GET都不会带有请求体。

http响应

响应报文包含：状态码，响应头，空行，响应正文。

  • 响应行：
    • 服务端的 HTTP 版本号、
    • 响应状态码
    • 状态说明
  • 响应头：请求头对应，是服务器对该响应的一些附加说明，比如响应内容的格式是什么，响应内容的长度有多少、什么时间返回给客户端的、甚至还有一些 Cookie 信息也会放在响应头里面。
  • 响应体：真正的响应数据，网页的 HTML 源代码。

二、Beautiful Soup

参考：Beautiful Soup 4.4.0 文档(中文版，部分翻译不准确)

原文档（bs4.9.0）

Beautiful Soup是一个可以从HTML或XML文件中提取数据的Python库.

解析器

不同的解析器可能影响代码执行结果，如果在分发给别人的代码中使用了 BeautifulSoup ，那么最好注明使用了哪种解析器，以减少不必要的麻烦。

解析器 使用方法 优势 劣势
Python标准库 BeautifulSoup(markup, "html.parser") 1.Python的内置标准库 2.执行速度适中 3.文档容错能力强 Python 2.7.3 or 3.2.2)前的版本中文档容错能力差
lxml HTML 解析器 BeautifulSoup(markup, "lxml") 1. 速度快 2. 文档容错能力强 需要安装C语言库
lxml XML 解析器 BeautifulSoup(markup, ["lxml-xml"])``BeautifulSoup(markup, "xml") 1. 速度快 2. 唯一支持XML的解析器 需要安装C语言库
html5lib BeautifulSoup(markup, "html5lib") 1. 最好的容错性 2.以浏览器的方式解析文档 3.生成HTML5格式的文档 1. 速度慢 2. 不依赖外部扩展

对象

Beautiful Soup将复杂HTML文档转换成一个复杂的树形结构，每个节点都是Python对象，所有对象可以归纳为4种：Tag ， NavigableString ， BeautifulSoup ， Comment 。

1. Tag

Tag最重要的两个属性

  • Name

  • Attributes

    属性的操作方法与字典相同

    如果转换的文档是XML格式，那么tag中不包含多值属性

2. NavigableString

NavigableString对象支持遍历文档树和搜索文档树中定义的大部分属性， 并非全部。尤其是，一个字符串不能包含其它内容(tag能够包含字符串或是其它tag)，字符串不支持 .contents 或 .string 属性或 find() 方法。

tag中包含的字符串不能编辑，但是可以被替换成其它的字符串，用replace_with()方法：

如果想在Beautiful Soup之外使用 NavigableString 对象，需要调用 unicode() 方法，将该对象转换成普通的Unicode字符串。否则就算你已经使用完Beautiful Soup，该对象的输出也会带有对象的引用地址（？your string will carry around a reference to the entire Beautiful Soup parse tree）。这样会浪费内存。

3.BeautifulSoup

BeautifulSoup对象表示的是一个文档的全部内容。大部分时候，可以把它当作 Tag对象。它支持遍历文档树和搜索文档树中描述的大部分的方法。

4.Comment（注释）

是特殊的NavigableString对象。

Beautiful Soup 为可能出现在 XML 文档中的任何其他内容定义了类：CData、ProcessingInstruction、Declaration 和 Doctype。 与 Comment 一样，这些类是 NavigableString 的子类，它们向字符串添加了一些额外的东西。
